## Применение обучения с подкреплением в задаче составления графиков смен городского общественного транспорта

### Дипломная работа, Гончаров Сергей, 4 курс 11 группа

Реализация задачи Crew Scheduling Problem (CSP) методами обучения с подкреплением. Два варианта агента: оффлайн (статическое расписание) и онлайн (расписание с возмущениями).

## Структура


## Задача

На вход подаётся расписание автобусного маршрута: контрольные пункты (КП), линии между ними, матрица времён холостых поездок и список отправлений с временными метками. Нужно назначить водителей на рейсы так, чтобы минимизировать число задействованных машин и суммарное время холостых поездок.

Агент принимает решение последовательно по каждому рейсу расписания: на каждом шаге ему предлагается набор из 8 кандидатов-водителей, он выбирает одного. Смена водителя ограничена 8 часами, обед обязателен через 4 часа работы.

## Онлайн-агент

Расширяет оффлайн-агент двумя механизмами:

**Возмущения** — интервальные задержки на линиях (например, +15 мин с 9:00 до 12:00). Конфигурируются в `src/config/online_scenario.yaml`.

**Планировщик холостых поездок** (`TimeWindowDeadheadPlanner`) — перед каждым шагом просматривает рейсы в окне 60 мин вперёд, используя оффлайн-политику для упреждающей отправки свободных автобусов к нужным КП.

Онлайн-агент инициализируется весами оффлайн-чекпоинта и дообучается на расписании с возмущениями.

## Конфигурация

Основные параметры среды (`src/config/environment.yaml`):

| Параметр | Значение | Описание |
|----------|---------|----------|
| `shift_duration` | 480 мин | длина рабочей смены |
| `work_duration_before_lunch` | 240 мин | работа до обеда |
| `bus_fleet_size` | 260 | размер парка |
| `target_bus_set_size` | 8 | ширина пространства действий |

Архитектура сети: MLP `[128, 64, 32]`, ReLU. Обучение: 1000 эпизодов, lr=1e-5, clip=0.1.

## Данные

Расписания двух городов в формате JSON:
- `data/schedule_obninsk.json` — Обнинск
- `data/schedule_yaroslavl.json` — Ярославль

Каждый файл содержит: список КП, матрицу холостых поездок, линии, расписание отправлений, маршруты.
